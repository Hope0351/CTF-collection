Hello All, this is a new series of blogs I am starting — Reverse Engineering. To start with this, we will see how to setup the lab to practice / gain knowledge on Reverse engineering ( a branch of CyberSecurity )

Simple definition of reverse engineering — Reverse-engineering is the act of dismantling an object to see how it works. It is done primarily to analyse and gain knowledge about the way something works but often is used to duplicate or enhance the object. Many things can be reverse-engineered, including software, physical machines, military technology and even biological functions related to how genes work.

- The idea of setting up a dedicated lab for Malware analysis / Reverse Engineering is to have a controlled / restricted environment for malware analysis / reverse engineering the malware.

- These steps expects some basic knowledge on how to use Oracle VM Virtual Box. I will try to put another blog detaining some basic information about Oracle VM Virtual Box. Until then, any questions, please put that in comments section or contact me directly , I can clarify these steps in detail.

-I am going to use the terms — host machine ( The physical machine — laptop / desktop ) and guest machine ( the VM we will build on top of our physical machine )
